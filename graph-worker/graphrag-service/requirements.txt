# Core (light) dependencies for the RabbitMQ consumer + health/metrics server.
#
# Deliberately does NOT pull in requirements-graphrag.txt: the service must be
# able to `pip install -r requirements.txt` and run the consumer loop (consume
# -> validate -> stub/process -> ack) with only these packages installed. The
# heavy GraphRAG/LLM pipeline (src/domain/graphrag, src/domain/ingestion, ...)
# is imported lazily at call time (see src/worker/processor.py) and only
# activates once requirements-graphrag.txt is installed *and* OPENAI_API_KEY
# is configured. Install requirements-graphrag.txt separately (or via the
# Dockerfile's INSTALL_GRAPHRAG build arg) to enable it.
